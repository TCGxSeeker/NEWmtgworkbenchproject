# View Modes Requirements

## Core View Modes

### Stacks

Show grouped cards in vertical piles or columns. Best for deckbuilding flow, category review, and quick card movement.

### Text

Show compact decklist text with quantities. Best for import/export review and fast scanning.

### Grid

Show visual card tiles with compact metadata. Best for browsing and recognizing cards.

### Table

Show dense rows with sortable columns. Best for data review, prices, ownership, mana value, type, and legality snapshots.

## Shared Requirements

- All modes use the same deck model.
- Changing view mode must not change deck data.
- Unknown cards and warnings must remain discoverable in every mode.
- Controls should not move unpredictably between modes.
