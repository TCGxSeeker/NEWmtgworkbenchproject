# Visual Reference Notes

## Purpose

Capture visual and interaction qualities from user-provided side-project
screenshots without importing their non-MTG domain, data, workflow rules, or
external-service behavior.

## Useful Visual Cues

- Strong top-level page framing with generous outer margins.
- A clear product title area that gives the screen confidence without crowding
  the working surface.
- Dark, quiet background with slightly lighter panels and subtle borders.
- Soft purple/blue action buttons with smooth rounded shapes.
- Secondary green success/status treatments that feel calm and readable.
- Tab-like mode switching that makes the active mode obvious.
- Bordered work sections that separate tasks without creating a busy dashboard.
- A side library/panel that is visually distinct from the main work area.
- Dense text areas can work when they are clearly framed as an editing surface.
- Buttons feel better when they have strong contrast, soft glow, and compact
  but comfortable padding.

## Interaction Cues

- Repeatable workflows benefit from obvious primary actions and stable panel
  placement.
- A collapsible or separate work surface for add/import/search behavior can
  feel familiar and efficient when it does not replace the main deck view.
- Status banners should confirm completed work clearly, then stay out of the
  way.
- A right-side library or context panel can support the main workflow when it
  remains secondary.

## Do Not Carry Forward

- Do not import non-MTG terminology, deck names, card names, file paths, or
  product-specific data from the reference.
- Do not copy the exact page, branding, colors, layout, or domain workflow.
- Do not add live-service import/update behavior from the reference.
- Do not use this reference to justify a search-first or utility-first main
  screen for MTG Workbench.
- Do not let dense text/editor surfaces crowd card understanding, card details,
  or deck validation later.

## MTG Workbench Translation

For future visual passes, use this reference as support for:

- polished dark panels
- satisfying button treatment
- calm success/status messages
- tidy task sections
- collapsible add/search surfaces
- a secondary library/details region

The MTG Workbench final UI should still feel like a modern deckbuilder
workbench, not a proxy utility clone.
