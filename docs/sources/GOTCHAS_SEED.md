# Gotchas Seed

## Purpose

Capture early project-specific mistakes, rejected assumptions, and recurring edge cases before they are promoted into `docs/codex/GOTCHAS.md`, tests, or skills.

## Fill In Later

- Misleading recommendation patterns.
- Incorrect role-counting assumptions.
- Bad scoring examples.
- UI copy or layout traps.
- Data import edge cases.
- User corrections that should become regression tests.
